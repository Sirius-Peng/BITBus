## ✨ 功能特性

### 🎯 核心功能

- **智能抢票**
    - ⚡ 自动在发车前 1 小时开始抢票
    - 🎲 支持手动选座和自动抢座两种模式
    - 🔄 自动轮询座位，直到抢到目标数量
    - 📊 支持座位优先级设置（高/中/低）

- **座位管理**
    - 🗺️ 可视化座位布局（真实巴士布局）
    - 👆 左键点击选座，右键设置优先级
    - 🎨 直观的座位状态显示（可选/已订/禁用）
    - 📈 批量设置座位优先级

- **任务管理**
    - 📝 任务列表实时更新
    - 🔍 任务状态追踪（准备中/等待开抢/抢票中/成功/失败）
    - ⏸️ 支持取消和删除任务
    - 📊 显示抢票进度和已抢座位

### 📢 通知系统

- **邮箱通知**
    - 📧 支持 SMTP 邮件发送
    - 🎨 HTML 格式邮件模板
    - ⚙️ 自定义 SMTP 服务器配置

- **企业微信通知**
    - 💬 企业微信群机器人
    - 📱 Markdown 格式消息
    - ⚡ 实时推送抢票结果

### 🛡️ 安全特性

- 🔐 配置信息本地存储
- 🔒 敏感信息脱敏显示
- ⚠️ 完善的错误处理机制
- 🚨 异常自动恢复

## 🚀 快速开始

### 环境要求

- Python 3.10 或更高版本
- pip 包管理器
- 稳定的网络连接

### 安装步骤

1. **克隆项目**

```bash
git clone https://github.com/LingRuo-BIT/BITBusGrab.git
cd BITs-BusGrab
```

2. **安装依赖**

```bash
pip install -r requirements.txt
```

3. **运行应用**

```bash
python app.py
```

4. **访问系统**

   打开浏览器访问：`http://localhost:23200`

### 获取 API 凭证

#### 方法一:自动捕获(推荐) 🌟

**工作原理：**

自动捕获功能基于本地HTTP代理实现，安全可靠：

1. **本地运行**：代理服务器运行在您的电脑上，不会将数据发送到任何外部服务器
2. **仅捕获必要信息**：只提取 API 请求头中的 `apitoken`、`apitime` 和 URL 参数中的 `userid`
3. **明文HTTP流量**：北理工后勤系统使用明文HTTP通信，代理可以直接读取
4. **即用即停**：捕获完成后立即关闭代理服务器，不会持续监控
5. **开源透明**：所有代码开源，您可以审查每一行代码

**技术细节：**

```
手机 → [代理服务器(本机:8888)] → 后勤系统服务器
         ↓
    捕获 HTTP 请求头:
    - apitoken: 用户身份令牌
    - apitime: 请求时间戳
    - userid: 用户ID
         ↓
    转发原始请求(不修改)
         ↓
    后勤系统正常响应
```

**隐私保护承诺：**

- ✅ 不记录任何其他数据（如密码、个人信息等）
- ✅ 不存储任何历史记录
- ✅ 不上传任何数据到互联网
- ✅ 代理服务器仅监听本地网络
- ✅ 捕获完成后自动关闭

**使用步骤：**

1. **启动自动捕获**
    - 点击系统设置中的"启动自动捕获"按钮
    - 系统会启动本地代理服务器（端口 8888）

2. **配置手机代理**
    - 确保手机和电脑在同一WiFi网络
    - 打开手机WiFi设置 → 选择已连接的WiFi
    - 配置HTTP代理:
        - 服务器:填写系统显示的IP地址
        - 端口:`8888`

3. **访问班车系统**
    - 在手机i北理工作台打开“班车服务”
    - 系统将自动捕获凭证

4. **自动完成**
    - 捕获完成后，凭证自动填充到配置中
    - 系统会弹出提醒，请立即关闭手机代理设置
    - 代理服务器自动关闭
    - ⚠️ **重要**：必须在手机上关闭代理设置，否则可能无法正常上网

**常见问题：**

**Q: 为什么需要代理？**
A: 北理工后勤系统使用HTTP明文传输，浏览器开发者工具在移动端不可用。代理是最简单、最安全的获取方式。

**Q: 代理会不会泄露我的隐私？**
A: 不会。代理服务器：

- 只运行在您的本机
- 只监听本地网络（不暴露到互联网）
- 只提取必要的API凭证
- 捕获完成立即关闭

**Q: 可以不使用代理吗？**
A: 可以选择手动抓包方式，但需要：

- 使用抓包工具（如Charles、Fiddler）
- 手动查找和复制参数

自动捕获方式更简单、更安全。

**Q: 捕获的凭证有效期多久？**
A: 根据测试，Token通常长时间不会失效。若失效，重新捕获即可。

#### 方法二:手动抓包

1. 使用抓包工具（如 Charles、Fiddler）监控手机流量
2. 登录北理工后勤系统班车服务页面
3. 查找请求头中的 `apitoken` 和 `apitime`
4. 查找 URL 参数中的 `userid`
5. 将获取的参数填写到系统配置中
6. 保存配置

## 📖 使用指南

### 基本操作

#### 1. 配置系统

首次使用需要配置 API 凭证：

1. 点击右上角 **设置** 按钮
2. 在 **API 配置** 标签页填写：
    - **API 地址**：`hqapp1.bit.edu.cn`（默认）
    - **API Token**：32 位字符（从抓包获取）
    - **API Time**：13 位时间戳（从抓包获取）
    - **用户 ID**：32 位字符（从抓包获取）
3. 点击 **保存配置**

#### 2. 查询班车

1. 选择 **线路方向**（如：良乡校区 → 中关村校区）
2. 选择 **日期**
3. 点击 **查询** 按钮
4. 查看可用班车列表

#### 3. 选择座位

1. 点击班车卡片上的 **选座** 按钮
2. 查看座位布局和可用座位
3. 左键点击选择座位
4. 点击 **确认抢票** 创建任务

### 座位优先级设置

#### 为什么需要优先级？

在自动抢票模式下，系统会按照优先级顺序尝试抢座，优先抢占高优先级座位。

#### 如何设置优先级？

**方法一：单个设置**

1. 右键点击任意座位
2. 在弹出菜单中选择优先级（高/中/低）
3. 座位上会显示优先级标识（H=高，L=低）

**方法二：批量设置**

1. 点击批量设置按钮
2. 选择 **全部高优先级** / **全部中优先级** / **全部低优先级**

**注意事项：**

- 默认所有座位为中优先级
- 已被预订的座位也可以设置优先级（用于候补）
- 优先级配置会保存在浏览器本地

### 自动抢票模式

#### 开启自动模式

1. 在选座页面勾选 **自动抢票模式**
2. 设置 **目标座位数量**（1-10 个）
3. 系统会按优先级自动抢座

#### 工作原理

```
开始抢票
    ↓
获取当前座位状态
    ↓
筛选可用座位
    ↓
按优先级排序
    ↓
尝试预订优先级最高的座位
    ↓
成功？
├─ 是 → 已抢座位数 +1 → 达到目标数量？
│                      ├─ 是 → 完成
│                      └─ 否 → 继续抢票
└─ 否 → 标记该座位为已尝试 → 继续抢票
```

### 通知配置

#### 邮箱通知

1. 在设置中切换到 **通知配置** 标签
2. 勾选 **启用邮箱通知**
3. 填写邮箱配置：
    - **SMTP 服务器**：如 `smtp.qq.com`
    - **端口**：通常为 `465`（SSL）或 `587`（TLS）
    - **发件邮箱**：您的邮箱地址
    - **授权码**：邮箱的 SMTP 授权码（非登录密码）
    - **收件邮箱**：接收通知的邮箱

**获取 QQ 邮箱授权码：**

1. 登录 QQ 邮箱
2. 设置 → 账户 → POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务
3. 开启 SMTP 服务
4. 生成授权码

#### 企业微信通知

1. 勾选 **启用企业微信通知**
2. 填写 **Webhook URL**

**获取 Webhook URL：**

1. 在企业微信群中添加群机器人
2. 复制机器人的 Webhook 地址
3. 粘贴到配置中

```json
  "API_TOKEN": "your-32-character-token",

  "USER_ID": "your-32-character-user-id",

    "email",
    "wechat_work"
  ],
  "email_config": {
    "smtp_server": "smtp.qq.com",
    "smtp_port": "465",
    "sender_email": "your@email.com",
    "sender_password": "your-smtp-auth-code",
    "receiver_email": "receiver@email.com"
  },
  "wechat_config": {
    "webhook_url": "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=..."
  }
}
```

优先级配置保存在 `seat_priorities.json` 中：

```javascript
{

        "4"
:
    2,   // 4 号座位：中优先级
        "5"
:
    3,   // 5 号座位：低优先级
    // ... 其他座位
}
```

## 📡 API 文档

### 后端 API 端点

#### 车辆查询

```http
POST /api/search
Content-Type: application/json

{
  "origin": "良乡校区",
  "destination": "中关村校区",
  "date": "2025-01-15"
}
```

#### 获取座位信息

```http
GET /api/seats/:bus_id?date=2025-01-15
```

#### 创建抢票任务

```http
POST /api/reserve
Content-Type: application/json

{
  "bus_id": "123456",
  "bus_info": {
    "origin_address": "良乡校区",
    "end_address": "中关村校区",
    "origin_time": "08:00",
    "date": "2025-01-15"
  },
  "seat_ids": [5, 6, 7],
  "auto_mode": true,
  "target_count": 2,
  "seat_priorities": {
    "5": 1,
    "6": 2,
    "7": 3
  }
}
```

#### 获取任务列表

```http
GET /api/tasks
```

#### 取消/删除任务

```http
POST /api/tasks/:task_id/cancel
DELETE /api/tasks/:task_id
```

#### 配置管理

```http
GET /api/config
POST /api/config
```

## ❓ 常见问题

### Q1: Token 过期怎么办？

**A:** Token 和 Time 是有时效性的，需要定期更新：

1. 重新登录后勤系统
2. 抓包获取新的 Token 和 Time
3. 在设置中更新配置

### Q2: 为什么抢票失败？

**可能原因：**

- ❌ Token 已过期
- ❌ 座位已被他人抢占
- ❌ 网络连接不稳定
- ❌ 系统维护中

**解决方案：**

1. 检查并更新 Token
2. 设置更多候补座位
3. 使用自动模式提高成功率

### Q3: 如何提高抢票成功率？

**建议：**

1. ✅ 使用自动抢票模式
2. ✅ 设置合理的座位优先级
3. ✅ 目标座位数设置为 1-2 个
4. ✅ 提前创建任务（系统会自动在合适时间开抢）
5. ✅ 候补已预订座位

### Q4: 邮箱通知发送失败？

**检查项：**

- ✅ SMTP 服务器地址是否正确
- ✅ 端口是否正确（QQ 邮箱用 465）
- ✅ 是否使用授权码而非登录密码
- ✅ 是否开启了 SMTP 服务

### Q5: 座位编号对应关系？

巴士座位布局（共 51 个座位）：

```
前方 →

[1]        [2]      (驾驶位旁，系统保留)
[3][4]   [5][6]
[7][8]   [9][10]
... (中间座位)
[47][48][49][50][51]  (最后一排)

← 后方
```

- 1、2 号：系统保留（司机旁）
- 49 号：系统保留
- 其他座位可选

### Q6: 可以同时抢多个时段的车吗？

**A:** 可以！系统支持创建多个抢票任务，每个任务独立运行。

### Q7: 配置会丢失吗？

**A:**

- API 配置保存在服务器 `config.json` 文件中，不会丢失
- 座位优先级保存在 `seat_priorities.json` 文件中，不会丢失
